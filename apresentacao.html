1. O Produto: APIVERSE - Marketplace de APIs

O objetivo do projeto foi desenvolver uma loja virtual para um produto incomum, explorando um nicho de mercado digital. A escolha foi criar o APIVERSE, um e-commerce focado na venda de APIs (Interfaces de Programação de Aplicações).

O que é uma API?

Uma API é um conjunto de rotinas e padrões que permitem a comunicação entre diferentes sistemas. Em vez de vender um produto físico, o e-commerce vende "pontes" de software que permitem que desenvolvedores e empresas adicionem funcionalidades complexas aos seus próprios sistemas de forma rápida e segura.

Por que é um Produto Incomum para um E-commerce?

A venda de APIs normalmente ocorre em plataformas B2B (Business-to-Business) altamente técnicas. A proposta do projeto é criar uma experiência de "varejo digital" para esse produto, com uma interface amigável, um "carrinho de compras" para licenças e um processo de checkout simplificado, tornando a aquisição de tecnologia tão fácil quanto comprar um produto online.

2. Modelo de Negócio e Lucratividade

O APIVERSE gera receita através de um modelo de negócio B2B, atuando como intermediário entre os criadores de APIs e os consumidores (desenvolvedores/empresas). As principais fontes de lucro são:

Comissão sobre Vendas

A principal fonte de receita. Para cada requisição paga que um cliente faz a uma API comprada no site (ex: mensalidade de R$ 30 por 1000 requisições), o APIVERSE retém uma porcentagem (ex: 20%) como comissão. Isso cria um fluxo de receita recorrente e escalável.

Taxa de Listagem e Destaque

Empresas que desejam anunciar e vender suas APIs na plataforma pagam uma taxa para ter seus produtos listados. Além disso, são oferecidos pacotes de "destaque", que posicionam a API na página inicial ou no topo das categorias, aumentando sua visibilidade e potencial de vendas.

Planos de Assinatura para Vendedores

Criadores de APIs podem optar por planos de assinatura (ex: Básico, Pro, Enterprise) que oferecem benefícios como taxas de comissão reduzidas, acesso a um painel de analytics avançado sobre o uso de suas APIs e suporte prioritário da equipe da plataforma.

3. Fases e Recursos do Projeto

O desenvolvimento do site foi planejado em quatro fases principais.

Fase 1: Descoberta e Planejamento

  • Atividades: Definição do escopo, pesquisa de mercado, identificação do público-alvo e criação da arquitetura da informação.
  • Ferramentas:Google Docs, Trello.

Fase 2: Design (UI/UX)

  • Atividades: Criação de wireframes, design do layout e prototipagem das interações do usuário.
  • Ferramentas: Figma, IA Geradora de Imagens.

Fase 3: Desenvolvimento (Front-end)

  • Atividades: Codificação da interface utilizando HTML semântico, estilização com CSS e implementação da interatividade com JavaScript.
  • Ferramentas: Visual Studio Code, HTML5, CSS3, JavaScript.

Fase 4: Simulação de Back-end e Lançamento

  • Atividades: A lógica de negócio foi simulada no cliente com JavaScript. A fase final seria o deploy em um servidor.
  • Ferramentas (simuladas): PHP (para o servidor), Hostinger (para hospedagem), Git/GitHub (para controle de versão).
